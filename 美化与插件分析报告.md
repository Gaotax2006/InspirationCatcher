# 灵感捕手 (Inspiration Catcher) — 布局美化与插件分析

## Context

对当前应用布局、CSS、FXML 进行全面审查，分析如何使界面更美观，以及推荐可集成的 JavaFX 插件/库。

---

## 一、当前布局结构

```
BorderPane
├── top: VBox
│   ├── MenuBar (文件/编辑/视图/工具/帮助 — 大量"未实现"项)
│   └── HBox Toolbar (项目选择器 + 字体控制 + AI测试 + 保存 + 标签管理)
├── center: SplitPane (divider=0.10/0.90)
│   ├── 左侧面板 (VBox): 快速捕捉按钮 / 搜索框 / 过滤器(Accordion) / 热门标签
│   ├── TabPane (灵感列表 | 思维导图 | 编辑器)
│   │   ├── 灵感列表: TableView (7列) + 顶栏按钮
│   │   ├── 思维导图: SplitPane (左侧灵感列表 + 右侧Canvas)
│   │   └── 编辑器: SplitPane (左侧表单+TextArea | 右侧WebView预览)
│   └── 右侧面板: VERTICAL SplitPane (灵感详情 / AI建议 / 相关灵感 / 统计)
└── bottom: HBox StatusBar
```

## 二、现有 CSS 状况

仅 3 个 CSS 文件，总计 ~300 行：
- **main.css**: 全局字体、几个按钮类（.quick-capture-btn, .save-btn, .ai-btn）
- **layout.css**: SplitPane 分割条样式、最小宽高约束
- **mindmap.css**: 思维导图节点阴影、连线样式、卡片样式

**问题**: 绝大多数样式使用 `style="-fx-..."` 内联方式写在 FXML 和 Java 代码中

## 三、当前布局的突出问题

### 3.1 视觉/UI 问题
1. **菜单栏充斥"未实现"占位项** — 文件/编辑/视图/工具/帮助 五个菜单大部分功能标注"未实现"，非常不专业
2. **右侧面板四个区域挤在一个垂直 SplitPane 里** — 灵感详情、AI建议、相关灵感、统计 四个区域在垂直空间内严重挤压
3. **颜色杂乱无体系** — 按钮颜色随机使用 #4A90E2、#FF9800、#4CAF50、#2196F3、#FF5722 等，没有统一的配色方案
4. **内联样式遍布** — FXML 和 Java 代码中大量 `style="-fx-..."`，CSS 文件几乎被架空
5. **emoji 代替图标** — "📝 快速捕捉"、"🤖 AI建议"、"📊 统计"、"➕ 灵感节点" 等，跨平台显示不一致
6. **左侧面板内容过多** — 项目选择器、搜索、6个筛选器、标签云 全部堆在一个窄面板里
7. **工具栏职责混杂** — AI测试按钮、字体大小控制、保存、清理标签放在同一行
8. **编辑器标签页布局低效** — 水平 50/50 SplitPane，编辑区和预览区都窄，尤其是属性表单（标题/类型/重要性/心情/标签）占用了大量垂直空间

### 3.2 功能/交互问题
1. **灵感详情和AI建议用 TextArea 显示** — 不支持富文本、不支持点击链接、样式单调
2. **统计信息放在右侧面板最底部** — 容易被忽略
3. **筛选器太多占空间** — 6 个筛选控件（类型/重要性/心情/开始日期/结束日期/标签）折叠在 Accordion 里
4. **没有空状态/加载状态** — TableView 为空时没有任何引导提示
5. **思维导图左侧面板重复** — 已经在主界面有灵感列表，在思维导图Tab里又显示了一份

## 四、推荐的 JavaFX 插件/库

### 4.1 核心 UI 库

| 库 | 用途 | 推荐等级 | 说明 |
|----|------|----------|------|
| **ControlsFX** | 增强控件库 | ⭐⭐⭐⭐⭐ | ToggleSwitch、RangeSlider、Notification、PopOver、TextField with autocomplete 等。对项目最有价值 |
| **Ikonli** | 图标字体库 | ⭐⭐⭐⭐⭐ | FontAwesome / Material Design Icons 等 30+ 图标集，完全替代 emoji |
| **AtlantaFX** | 现代主题 | ⭐⭐⭐⭐ | FlatLaf 风格的 JavaFX 主题，统一外观，无需手写大量 CSS |

### 4.2 功能增强库

| 库 | 用途 | 推荐等级 | 说明 |
|----|------|----------|------|
| **RichTextFX** | 代码编辑器 | ⭐⭐⭐ | 替换 Markdown 编辑器的 TextArea，支持语法高亮和行号 |
| **AnimateFX** | 动画库 | ⭐⭐ | 为按钮/面板添加过渡动画，提升交互感，非必需 |
| **GemFX** | 现代 UI 控件 | ⭐⭐ | 类似 Fluent Design，但与 JavaFX 21 兼容性有待验证 |

### 4.3 不需额外库即可实现的改进

| 改进方式 | 说明 |
|----------|------|
| **WebView 内嵌 CSS** | 利用 MarkdownUtil 的 WebView 渲染更多样式 |
| **CSS 伪类** | `:hover` `:focused` `:selected` 等替代内联鼠标事件样式 |
| **CSS 变量** | JavaFX 不支持 CSS 变量，但可以用 Looked-up Colors `-fx-my-color` 实现主题一致性 |
| **JavaFX SVGPath** | 用 SVG 图标替代 emoji，无需引入外部库 |

## 五、实施方案（一次性全面美化）

### Step 1: 添加依赖
**修改 `pom.xml`**，新增：
```xml
<!-- ControlsFX - 增强控件 -->
<dependency>
    <groupId>org.controlsfx</groupId>
    <artifactId>controlsfx</artifactId>
    <version>11.2.0</version>
</dependency>
<!-- Ikonli - 图标库 -->
<dependency>
    <groupId>org.kordamp.ikonli</groupId>
    <artifactId>ikonli-javafx</artifactId>
    <version>12.3.1</version>
</dependency>
<dependency>
    <groupId>org.kordamp.ikonli</groupId>
    <artifactId>ikonli-fontawesome5-pack</artifactId>
    <version>12.3.1</version>
</dependency>
<!-- AtlantaFX - 现代主题 -->
<dependency>
    <groupId>io.github.mkpaz</groupId>
    <artifactId>atlantafx-base</artifactId>
    <version>2.0.1</version>
</dependency>
```

### Step 2: CSS 全面重构
- 创建 `css/theme.css` — 定义颜色变量、全局圆角、阴影、间距系统
- 创建 `css/components.css` — 按钮、卡片、表单、标签的统一样式
- 重写 `css/main.css` — 精简为基础布局和覆盖
- 移除 `layout.css` — 合并到 theme.css
- 废弃 `mindmap.css` — 合并到 components.css
- 从 FXML 和 Java 中提取所有内联样式到 CSS 文件

### Step 3: 布局重构
- **清理菜单栏**: 删除所有"未实现"的占位菜单项
- **右侧面板**: 垂直 SplitPane 的4个区域改为 TabPane（详情/AI/相关/统计），或把统计合并到状态栏
- **编辑器**: 属性表单（标题/类型/重要性/心情/标签）合并到 Markdown 工具栏上方，减少垂直占用
- **筛选器**: 日期筛选合并为一行，移除不必要的折叠面板
- **思维导图**: 移除左侧重复的灵感列表面板，只保留画布和工具栏

### Step 4: Ikonli 替换 emoji
- 所有按钮、Label、标题中的 emoji 替换为 FontAwesome 图标
- 工具栏图标化（B/I/H1/H2/代码/列表/引用/链接/图片）
- 类型图标（💡📝❓✅ 等）替换为对应 FontAwesome 图标
- 心情图标（😊🎉😌 等）替换为对应 FontAwesome 图标

### Step 5: AtlantaFX 主题集成
- 在 `MainApp.java` 中应用 AtlantaFX 主题（`Application.setUserAgentStylesheet(...)`）
- 选择深色或浅色变体
- 利用 AtlantaFX 内置的现代控件样式

### Step 6: 交互增强
- 使用 ControlsFX Notifications 替代 Alert 提示
- 空状态/加载状态的全屏引导卡片
- 思维导图节点选中/悬停动画

## 七、验证方法

1. **编译**: `mvn compile` 确保依赖添加正确
2. **功能检查**: 启动应用后所有按钮/菜单功能正常
3. **视觉检查**: 布局比例合理，无重叠/截断，颜色统一
4. **图标检查**: Ikonli 图标正确渲染，无乱码
5. **分辨率适配**: 调整窗口大小，检查布局伸缩
